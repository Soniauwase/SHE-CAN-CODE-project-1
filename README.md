# Git and GitHub Fundamentals Assignment

## Overview
This assignment demonstrates how to use Git and GitHub to track changes, manage branches, and collaborate on a project using modern development tools.

## How I Used Git and VS Code

For this assignment, I used Git for version control and Visual Studio Code (VS Code) as my code editor.

First, I created a new project folder and initialized a Git repository using the `git init` command. I used the terminal inside VS Code to run all Git commands, which made it easier to manage everything in one place.

I created a README.md file and edited it directly in VS Code. VS Code helped me write and format the file using Markdown, allowing me to clearly organize titles, sections, and paragraphs.

I then staged my changes using `git add` and saved them using `git commit`. This helped me track the progress of my work step by step.

To practice branching, I created a new branch called `feature` using Git. In this branch, I added a new file and committed the changes. After that, I switched back to the main branch and merged the feature branch into it.

Finally, I connected my local repository to GitHub using `git remote add origin` and pushed my project online using `git push`. This allowed me to store my project in a remote repository.

## What This Assignment Demonstrates About GitHub

This assignment shows that GitHub is a powerful platform for hosting and managing code online. It allows developers to:

- Store and back up their projects in remote repositories
- Track changes and maintain version history
- Collaborate with others using branches and pull requests
- Review code and suggest improvements before merging changes

Using GitHub, I was able to simulate real-world collaboration by creating a pull request, adding comments, and merging changes into the main branch.

## Conclusion
Overall, this assignment helped me understand how Git and GitHub work together with VS Code to support efficient coding, version control, and collaboration in software development.
